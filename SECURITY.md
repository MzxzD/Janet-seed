# 🔐 Security & Privacy — J.A.N.E.T. Seed

Security in J.A.N.E.T. Seed is not just about exploits.  
It is about **trust, restraint, and respect for human boundaries**.

---

## 🧭 Security Philosophy

J.A.N.E.T. Seed is designed to be:
- Offline-first
- Consent-driven
- Minimally invasive
- Transparent in behavior

If a security improvement would require:
- silent data collection
- background network activity
- reduced user awareness

…it is not acceptable here.

---

## 🧠 Memory & Privacy

Janet treats memory as **sacred**.

- Secrets are never stored by default
- Memory writes are gated by constitutional rules
- Episodic memory is encrypted
- Semantic memory is scoped and inspectable
- Users can inspect, forget, or disable memory

Any change affecting memory **must be reviewed with extra care**.

---

## 🚨 Reporting a Vulnerability

Security issues should be reported **publicly and transparently**.

Please open a **Security Vulnerability Issue** using the dedicated issue template.

When reporting, include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact on privacy, safety, or integrity
- Whether the issue affects memory, consent, or interruptibility

If you believe immediate harm is possible, please clearly mark the issue as **URGENT**.

We value openness, calm collaboration, and shared responsibility.
Public discussion helps the project — and its users — stay safer.

---

## 🚫 What We Will Not Do

- Collect telemetry
- Phone home
- Obfuscate behavior
- Hide security trade-offs

Transparency is part of security.

---

## 🌱 Responsible Handling

Security discussions will be handled with care and respect.

The goal is to:
- understand the issue
- protect users
- preserve trust

Not to assign blame.

Thank you for helping keep Janet safe.
