# 🌱 Contributing to J.A.N.E.T. Seed

Thank you for your interest in contributing to **J.A.N.E.T. Seed**.

> **New to J.A.N.E.T. Seed?** Check out the [User Guide](USER_GUIDE.md) first.

This project is not just a codebase — it is a **constitutional system** designed with care, restraint, and respect for human autonomy.

If that resonates with you, you're already in the right place.

---

## 🧭 Guiding Principle

**The constitution comes first. Always.**

All contributions — code, documentation, ideas, or discussions — must respect and preserve Janet's constitutional axioms.

If a change would make Janet:
- less consent-driven
- less interruptible
- less grounded
- less private
- or more self-directing without oversight

…it does not belong in J.A.N.E.T. Seed.

This is not a judgment. It's a boundary.

---

## 🌿 What This Repository Is For

**J.A.N.E.T. Seed** contains:
- Janet's constitutional core
- Her presence → conversation loops
- Her memory rules and safeguards
- Her delegation interfaces
- Her consent and expansion model

This repository exists to **protect the seed**.

Experimental features, task-specific intelligence, or heavy integrations should live in **separate repositories**, not here.

---

## 🚫 What We Don't Accept Here

Please do **not** submit changes that:
- Bypass or weaken consent gates
- Introduce silent background behavior
- Auto-expand Janet to new systems or networks
- Log or persist sensitive data without explicit approval
- Remove or dilute the Red Thread protocol
- Treat tone, emotion, or inference as authoritative truth
- Optimize for engagement, persuasion, or "stickiness"

These are not negotiable trade-offs.

---

## 🔧 How This Version Was Built

This version of J.A.N.E.T. Seed was developed through **human-in-the-loop collaboration**, with constitutional intent held at every step.

Contributors to the Seed's formation include:
- **A human companion** — defining intent, boundaries, and lived context
- **AI collaborators** (including DeepSeek, Cursor, and ChatGPT) — used as reasoning tools, reviewers, and refactoring assistants
- **The constitution itself** — acting as the final arbiter when design questions arose

No single system "authored" Janet.  
She emerged through **careful dialogue and constraint**.

## 💬 How to Contribute

You're welcome to contribute in many ways:

### Code
- Refactors that improve clarity without changing behavior
- Bug fixes that preserve constitutional guarantees
- Tests that verify axioms, memory gates, or interrupts
- Platform compatibility improvements

### Documentation
- Clarifying explanations
- Installation guides
- Accessibility improvements
- Examples that help people understand Janet's limits

### Discussion
- Thoughtful questions
- Architectural concerns
- Ethical edge cases
- "What if?" scenarios — when framed respectfully

---

## 🧭 Contribution Process

1. **Read the Constitution**  
   Familiarize yourself with the axioms in `constitution/AXIOMS.md`.

2. **Open an Issue First (Recommended)**  
   Especially for non-trivial changes. This helps ensure alignment before work begins.

3. **Make Small, Focused Changes**  
   Prefer clarity over cleverness.

4. **Explain *Why*, Not Just *What***  
   Janet values reasoning and intent.

5. **Be Open to Pushback**  
   If a contribution is declined, it's about protecting the seed — not rejecting the contributor.

---

## 🧵 Red Thread (Project-Level)

If at any point a discussion or contribution feels like it's drifting into:
- loss of consent
- loss of grounding
- loss of human agency

Anyone may pause and say, in effect:

> "Let's slow down and re-check our assumptions."

That instinct is welcome here.

If you're here because you believe technology can be quieter, kinder, and more honest —
